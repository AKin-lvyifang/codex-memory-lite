# Codex 前端统一设计标准

## 这份文档是干什么的

这是一份给 Codex 用的统一前端标准。

目的不是教人画设计稿，而是让 Codex 在只拿到自然语言需求时，也尽量按同一套组件、颜色、排版和页面气质去落地。

## 使用规则

默认先阅读：

- `.codex-memory/spec/frontend-design-standards.md`

再开始设计或实现页面。

如果用户没有特别说明，就不要更换组件体系，也不要临时发明新的视觉语言。

## 默认组件体系

- 默认统一使用 `shadcn/ui`
- 不要混用第二套大型组件库
- 如果当前项目技术路线不适合直接使用 `shadcn/ui`，应先明确说明，再决定是调整路线还是只沿用视觉规则

## 默认视觉语言

- 白底
- 中性色为主
- 低噪音
- 红色只打重点
- 产品化、干净、克制

不要随意做成营销站或过度炫技页面。

## 默认排版纪律

- 小说明 / 标签：`14px`
- 正文：`16px`
- 区块标题：`20px`
- 页面标题：`24px`
- 关键数字：`30px`

## 默认圆角纪律

- 小控件：`8px`
- 卡片 / 大容器：`14px`
- Pills：`9999px`

## 默认颜色纪律

- 背景：`#ffffff`
- 主文字：`#0a0a0a`
- 弱文字：`#737373`
- 边框：`#e5e5e5`
- 主强调色：`#d41616`

## 说明

如果你还有团队自己的视觉规范文档，可以把它们作为补充参考，但本文件才是给 Codex 执行时使用的标准版。
